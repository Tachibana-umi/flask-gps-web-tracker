# 位置追踪 Web 项目 · 目标清单

---

## 一、后端（Flask）

### 核心接口
- [ ] 接收前端上传的位置数据（POST `/api/v1/location`）
- [ ] 返回历史位置记录，支持分页（GET `/api/v1/location`）
- [ ] 计算两点间距离、累计总路程等基础数据处理

### 安全与鉴权
- [ ] 实现用户注册 / 登录接口
- [ ] 集成 JWT 认证，所有数据接口均需验证 token
- [ ] 对所有接口输入做合法性校验（经纬度范围、字段类型等）
- [ ] 配置 CORS，明确限定允许访问的来源域名
- [ ] 接口速率限制（Rate Limiting），防止恶意频繁调用

### 工程规范
- [ ] 使用 `.env` 文件管理密钥、数据库连接等敏感配置，不硬编码进代码
- [ ] 接入日志系统，记录关键操作与错误信息（替代 `print`）
- [ ] 接口路径统一加版本前缀 `/api/v1/`，便于后续迭代
- [ ] 集成 Flask-Admin，提供后台数据管理界面

---

## 二、数据库

- [ ] 设计位置记录表：经度、纬度、时间戳（UTC）、用户 ID 等字段
- [ ] 对用户 ID 和时间戳字段建立数据库索引
- [ ] 实现软删除（`is_deleted` 字段），不物理删除数据
- [ ] 所有时间统一以 UTC 存储，前端显示时转换为本地时区
- [ ] 开发阶段使用 SQLite；上线后迁移至 PostgreSQL（+ PostGIS 空间扩展）

---

## 三、前端

### 位置获取与上传
- [ ] 调用浏览器 Geolocation API 获取 GPS 坐标
- [ ] 实现位移阈值或定时上传策略（如移动 >10 米 或每 5 秒才上传一次），避免无效请求
- [ ] 断网时将失败数据暂存至 localStorage / IndexedDB，网络恢复后自动重传

### 地图可视化
- [ ] 集成 Leaflet.js，渲染实时位置标记
- [ ] 展示历史轨迹连线
- [ ] 展示统计信息（总路程、打点数量等）

### 用户体验
- [ ] 每个网络请求均有 Loading 状态反馈
- [ ] 请求失败时显示友好的错误提示，而非空白页面
- [ ] 页面适配移动端（响应式布局），按钮与控件符合触屏操作习惯
- [ ] 前端对发送数据做基础格式校验，减少无效请求

---

## 四、部署与访问

- [ ] 配置 HTTPS（手机浏览器调用 GPS 的强制要求）
- [ ] 开发阶段使用 ngrok 做内网穿透，供手机真机测试
- [ ] 上线阶段部署至云服务器（如腾讯云 / 阿里云轻量应用服务器）
- [ ] 使用环境变量区分开发环境与生产环境配置

---

## 五、可选进阶目标

- [ ] 支持多用户，数据严格按用户隔离
- [ ] 按时间段筛选轨迹（今天 / 本周 / 自定义范围）
- [ ] 利用 PostGIS 实现半径范围查询（如"查询某地点附近的打点"）
- [ ] 轨迹导出为 GPX / CSV 文件
- [ ] 前端 PWA 化，支持添加到手机桌面并在弱网环境下正常使用
